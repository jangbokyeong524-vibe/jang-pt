# PT 회원 예약 MVP 운영 매뉴얼

이 문서는 MVP 기준 운영 순서를 설명한다. MVP는 체육관 전체 관리앱이 아니라 PT 회원권, 예약, 결제상태, 차감 판단을 관리하는 운영판이다.

## 1. 매일 시작할 때

1. 앱을 모바일에서 연다.
2. 관리자 화면으로 들어간다.
3. `처리 필요` 패널을 먼저 확인한다.
4. 오늘/이번 주 주간 시간표를 확인한다.

처리 우선순위:

1. 오늘 수업 관련 취소요청
2. 예약요청
3. 결제확인 필요
4. 회원연결 승인
5. 연장요청
6. 재등록 대상

하루 기본 순서:

1. 처리 필요를 비운다.
2. 오늘 확정 수업을 확인한다.
3. 수업이 끝난 뒤 수업완료와 차감을 기록한다.
4. BOX POS 결제 확인 후 앱 결제상태를 맞춘다.
5. 연장요청을 승인 또는 거절하고 기록을 확인한다.
6. 재등록 대상에게 연락할 회원을 확인한다.

## 2. 예약 요청 처리

회원이 예약을 요청하면 처리 필요 패널에 표시된다.

승인할 때:

1. 요청 회원과 시간을 확인한다.
2. 슬롯이 실제로 가능한지 확인한다.
3. `승인`을 누른다.
4. 필요하면 카카오톡 복사용 문구를 보내 확정 안내를 한다.

거절할 때:

1. 요청 시간에 수업이 어려운 이유를 확인한다.
2. `거절`을 누른다.
3. 회원에게 가능한 다른 시간을 안내한다.

기준:

- 요청 상태는 기본 24시간 유지된다.
- 만료된 요청은 다시 예약 요청을 받는다.

## 3. 수업 완료 처리

수업이 실제로 끝난 뒤 처리한다.

1. 관리자 주간 시간표에서 해당 확정 수업을 찾는다.
2. `수업완료`를 누른다.
3. 잔여횟수 1회 차감 여부를 확인한다.
4. 외부 장부에 옮길 필요가 있으면 설정에서 CSV 내보내기를 사용한다.

주의:

- 예약이 있었다는 이유만으로 자동 차감하지 않는다.
- 실제 수업이 완료된 경우에만 완료 처리한다.
- 앱은 중복 클릭으로 두 번 차감되지 않도록 설계한다.

## 4. 취소 처리

24시간 전 취소:

- 회원이 직접 취소할 수 있다.
- 차감하지 않는다.
- 슬롯은 다시 예약 가능 상태가 된다.

24시간 이내 취소:

1. 처리 필요 패널에서 취소요청을 확인한다.
2. 사유를 확인한다.
3. 기본 원칙은 차감이다.
4. 질병, 부상 등 예외 사유가 있으면 미차감 처리할 수 있다.

기록 기준:

- 차감/미차감 판단은 반드시 앱에 남긴다.
- 민감한 건강정보는 자세히 적지 않는다.
- 예: `부상으로 미차감`, `질병으로 7일 연장` 정도로 기록한다.

## 5. PT권 등록

신규 PT권 등록:

1. 관리자 화면에서 `회원/PT권` 탭을 연다.
2. 회원을 선택한다.
3. PT권 상품을 선택한다.
4. 앱에 새 PT권이 등록된다.
5. 기본 상태는 미납으로 둔다.

기준:

- PT권은 1회~10회권을 기본으로 한다.
- 기본 단가는 1회 60,000원이다.
- 1~5회권은 기본 30일, 6~10회권은 기본 60일이다.
- 상품 가격과 유효기간은 운영 설정에서 바꿀 수 있다.

## 6. BOX POS 결제 처리

BOX POS 원격 결제:

1. 앱에서 회원 전화번호와 금액을 확인한다.
2. BOX POS에서 금액과 휴대폰번호를 입력한다.
3. BOX POS가 회원에게 문자 결제 링크를 발송한다.
4. 앱 결제 상태를 `결제요청`으로 바꾼다.
5. BOX POS에서 결제 완료를 확인한다.
6. 앱 결제 상태를 `결제완료`로 바꾼다.

환불:

1. 미사용 횟수와 기본 환불액을 확인한다.
2. 실제 환불액이 다르면 메모를 남긴다.
3. BOX POS 또는 외부 방식으로 환불 처리한다.
4. 앱 결제 상태를 `환불`로 바꾼다.

주의:

- 앱은 실제 결제를 수행하지 않는다.
- 앱은 결제상태와 이력만 관리한다.

## 7. 회원 연결 승인

회원이 로그인 후 전화번호를 입력하면 연결 요청이 생긴다.

처리:

1. 처리 필요 패널에서 회원연결 요청을 확인한다.
2. 입력 전화번호와 기존 회원 전화번호를 비교한다.
3. 이름 또는 카카오톡 대화 내용과 맞는지 확인한다.
4. 맞으면 승인한다.
5. 애매하면 승인하지 않고 카카오톡/전화로 확인한다.

주의:

- 전화번호 직접 입력은 본인인증이 아니다.
- 잘못 승인하면 다른 회원의 PT권/결제상태가 노출될 수 있다.

## 8. 연장 요청 처리

회원이 연장을 요청하면 처리 필요 패널에 표시된다.

처리:

1. 사유를 확인한다.
2. 요청 일수를 확인한다.
3. 승인하면 만료일이 늘어난다.
4. 승인 후 연장 요청 상태와 차감/연장 이력에 기록이 남았는지 확인한다.
5. 거절하면 사유를 카카오톡으로 안내한다.

기준:

- 질병/부상/출장/개인사정은 기본 사유로 둔다.
- 운영상 예외는 관장이 판단한다.
- 승인 시 PT권 만료일 변경과 연장 승인 이력이 함께 남아야 한다.
- 같은 요청이 중복 승인되어 만료일이 두 번 늘어나면 안 된다.
- 자세한 의료정보는 앱에 적지 않는다.

## 9. 재등록 유도

기본 기준:

- 잔여 2회 이하
- 또는 만료 7일 전

처리:

1. 처리 필요 패널에서 재등록 대상을 확인한다.
2. 회원의 최근 출석과 남은 예약을 확인한다.
3. 카카오톡 복사용 문구를 활용해 상담한다.
4. 재등록하면 새 PT권을 등록한다.

## 10. CSV 내보내기와 비타민CRM 정리

앱은 비타민CRM 자동 연동을 하지 않는다.

정리 순서:

1. 관리자 화면에서 `설정` 탭을 연다.
2. `CSV 내보내기`에서 필요한 항목을 선택한다.
3. 전화번호가 필요한 작업이면 `개인정보 포함`을 켠다.
4. 다운로드 버튼으로 CSV 파일을 저장한다.
5. 비타민CRM 또는 외부 장부에 필요한 내용을 옮긴다.

선택 가능 항목:

- 회원
- 회원 연결 요청
- PT권
- 차감/연장 이력
- 예약 슬롯
- 예약
- 결제
- 결제 변경 이력
- 연장 요청
- PT 상품
- 운영 정책

주의:

- `개인정보 포함`을 켜지 않으면 전화번호 필드는 제외된다.
- CSV 파일은 `dataset, record_id, field, value` 기준의 긴 행 포맷으로 저장된다.

## 11. 운영 설정 변경

설정 변경 가능 항목:

- PT 상품 가격
- 기본 유효기간
- 예약 공개 주차
- 요청 만료 시간
- 자동취소 기준
- 당일취소 기본 차감 여부
- 연장/취소 사유 목록
- 재등록 기준
- 카카오톡 복사용 문구

원칙:

- 정책 변경은 신규 PT권/신규 예약부터 적용한다.
- 기존 PT권과 예약은 생성 당시 정책 스냅샷을 유지한다.
- 보안, 권한, 중복차감 방지 규칙은 설정으로 끄지 않는다.

## 12. MVP 밖 업무

다음 업무는 이번 MVP 운영 매뉴얼의 범위가 아니다.

- 그룹수업 출석부
- 일반 체육관 회원권
- 락커 관리
- 물품 판매
- 전체 정산 대시보드
- 카카오톡 자동발송
- PG 결제 또는 BOX POS API 연동

이 항목들은 PT 회원 예약 MVP가 실제로 안정화된 뒤 별도 제품 범위로 다룬다.
