# 강동무에타이장 PT 관리 앱 v1 UI/UX 디자인 문서

## 1. 디자인 방향

- 모바일 접속을 1순위로 둔다.
- 관장과 회원 모두 휴대폰에서 가장 자주 접속한다고 전제한다.
- UI는 모바일 화면에서 먼저 성립해야 하며, 태블릿/PC에서는 같은 구조를 넓게 확장한다.
- 현장 운영 도구이므로 장식보다 빠른 확인, 빠른 판단, 빠른 처리를 우선한다.
- 마케팅형 랜딩 페이지는 만들지 않는다. 첫 화면은 바로 사용할 수 있는 운영 화면이어야 한다.

## 2. 모바일 우선 기준

- 모든 주요 버튼과 탭은 최소 44px 이상의 터치 영역을 가진다.
- 모바일에서 페이지 전체 가로 스크롤은 생기면 안 된다.
- 가로 스크롤은 관리자 탭, 회원 선택 목록처럼 의도된 영역에만 허용한다.
- 카드 안의 텍스트는 줄바꿈되어야 하며, 버튼이나 상태 배지를 침범하면 안 된다.
- 모바일 첫 화면에서 `처리 필요`와 오늘/이번 주 예약 상태가 바로 보여야 한다.
- PC 전용 기능은 만들지 않는다. 모든 주요 기능은 모바일에서 완료 가능해야 한다.

## 3. 관리자 UX

- 관리자 모바일 첫 흐름은 `처리 필요 확인 -> 주간 시간표 확인 -> 예약/취소/수업완료 처리`이다.
- 처리 필요 패널은 모바일에서 가장 먼저 노출한다.
- 처리 필요 항목은 종류, 회원명, 금액/횟수 등 판단에 필요한 최소 정보만 보여준다.
- 주간 시간표는 날짜별로 세로 배치한다.
- 슬롯 상태는 `가능`, `요청중`, `확정`, `차단`을 명확히 구분한다.
- 액션 버튼은 아이콘 중심으로 두되, `title`/`aria-label`로 의미를 제공한다.

## 4. 회원 UX

- 회원 모바일 첫 흐름은 `내 PT 상태 확인 -> 예약 가능 시간 확인 -> 예약 요청/취소`이다.
- 회원 화면 상단에는 잔여횟수, 만료일, 결제상태, 재등록 상태를 요약한다.
- 예약 가능 시간은 한 번에 훑기 쉬운 세로 목록으로 제공한다.
- 회원이 예약 요청을 보낸 뒤에는 승인 대기 상태가 명확히 보여야 한다.
- 결제상태가 미납이거나 결제요청 상태면 회원 화면에서 숨기지 않고 명확히 표시한다.

## 5. 정보 구조

- 관리자 화면의 기본 구조:
  - 상단: 앱 이름, 관리자/회원 전환
  - 상태줄: 최근 처리 결과
  - 처리 필요 패널
  - 관리자 탭
  - 주간 시간표, 회원/PT권, 운영 설정, CRM 요약
- 회원 화면의 기본 구조:
  - 계정 선택/승인 상태
  - 내 PT 요약
  - 내 예약
  - 예약 가능 시간

## 6. 시각 규칙

- 카드 반경은 8px 이하로 유지한다.
- 상태 배지는 색으로 빠르게 구분하되 텍스트 라벨을 반드시 함께 제공한다.
- 주요 상태 색상:
  - 정상/완료: 녹색 계열
  - 요청/미납/주의: 노란색 계열
  - 취소요청/위험: 붉은색 계열
  - 정보: 파란색 계열
- 한 가지 색상만 지배하는 화면은 피한다.
- 화면 섹션을 과도한 중첩 카드로 만들지 않는다.

## 7. 반응형 기준

- 기본 CSS는 모바일 레이아웃이다.
- `min-width: 721px`부터 태블릿/작은 PC 확장 레이아웃을 적용한다.
- `min-width: 1101px`부터 데스크톱 다열 레이아웃을 적용한다.
- 모바일에서는 관리자 탭과 회원 선택 목록을 가로 스크롤 영역으로 둔다.
- 데스크톱에서는 처리 필요 패널을 좌측 sticky 영역으로 둔다.
