# 강동무에타이장 PT 회원 예약 MVP SPEC

## 0. 현재 구현 상태

- 현재 저장소는 `로컬 데모 UI`와 `Supabase 목표 설계`가 함께 들어 있는 초안 단계다.
- 화면은 `lib/seed-data.ts`의 데모 데이터와 클라이언트 상태로 동작한다.
- Kakao/Google 로그인, Supabase RLS, DB 읽기/쓰기, 예약 요청/취소/수업완료 DB 트랜잭션은 아직 화면에 완전히 연결되지 않았다.
- 이 문서는 먼저 사용할 MVP 기준을 정의한다. 구현 작업에서는 현재 로컬 데모 상태와 이 MVP 목표 상태를 구분해서 판단한다.

## 1. 제품 목적

- MVP는 강동무에타이장의 PT 운영을 실제로 돌릴 수 있는 최소 제품이다.
- PT권, 결제상태, 잔여횟수, 예약 요청, 예약 승인/거절, 취소, 수업완료 차감, 당일취소 차감/미차감 판단을 새 앱 기준으로 관리한다.
- 회원은 본인 PT권 상태를 확인하고 직접 예약을 요청/취소할 수 있어야 한다.
- 관장은 회원 연결, PT권 등록, 결제상태 수기 관리, 예약 처리, 차감 처리를 모바일에서 완료할 수 있어야 한다.
- 비타민CRM은 기존 전체 회원/정산 병행 장부로 유지한다.
- 새 앱은 데이터 보존과 외부 장부 이관을 위한 설정 기반 CSV 내보내기를 제공한다.
- 기술 구성은 `Next.js + Supabase Auth/DB + Vercel`을 기준으로 한다.

## 1.1 MVP 범위

MVP에 포함한다:

- 관리자: 회원 연결 승인, PT권 등록, 결제상태 수기 변경, 예약 승인/거절, 수업완료 차감, 24시간 이내 취소 차감/미차감 처리, 재등록 대상 확인, CSV 내보내기
- 회원: 로그인 후 승인대기, 본인 PT권/결제상태 확인, 예약 가능 시간 확인, 예약 요청, 예약 취소/취소요청, 예약/결제/차감 이력 확인
- 정책: 예약 공개 주차, 요청 만료 시간, 예약 개수 제한, 미납 예약 허용 여부, 자동취소 기준, 당일취소 기본 차감 여부, 재등록 기준
- 데이터 안전장치: RLS, 관리자/회원 권한 분리, 예약 RPC, 수업완료/당일취소 중복 차감 방지, 결제상태 변경 이력

MVP에서 제외한다:

- 그룹수업, 일반 체육관 출석부, 전체 회원권 관리, 락커, 물품 판매, 전체 정산 대시보드
- SMS 본인인증, 카카오톡 자동발송 API, PG 결제, BOX POS API 연동, 비타민CRM 자동 연동
- 복수 지점/복수 관리자 권한 체계

체육관 전체 관리앱 확장은 MVP 이후 방향으로만 다룬다. MVP 구현 중에는 PT 회원 예약 흐름을 우선한다.

## 2. 핵심 설계 원칙

- 모바일 접속을 1순위로 설계한다.
- 관장과 회원 모두 휴대폰에서 가장 자주 접속한다고 전제한다.
- 기본 화면, 정보 밀도, 버튼 크기, 처리 순서는 모바일 기준으로 먼저 결정한다.
- PC 화면은 별도 제품이 아니라 모바일 기준 구조를 넓게 확장한 화면이다.
- PC 전용 핵심 기능은 만들지 않는다. 모든 주요 기능은 모바일에서 완료 가능해야 한다.

## 3. 사용자와 권한

- 관리자: 관장 1명 운영 기준으로 설계한다.
- 회원: Kakao/Google 로그인 후 전화번호를 직접 입력한다.
- 전화번호 직접 입력은 본인인증이 아니라 회원 매칭 힌트로 취급한다.
- 관장이 회원 연결을 승인해야 회원이 본인 PT권, 예약, 결제상태를 볼 수 있다.
- 승인 전 회원은 승인대기 상태만 볼 수 있고 PT 운영 데이터는 볼 수 없다.
- 현재 로컬 데모 UI의 관리자/회원 전환은 개발과 테스트 확인용이다.
- 최종 제품에서는 로그인 계정의 권한에 따라 관리자 화면 또는 회원 화면으로 분기한다.

## 4. 관리자 기능

- 관리자 첫 화면은 PT 운영 `처리 필요 + 주간 요약`이다.
- 처리 필요 패널에는 예약요청, 취소요청, 연장요청, 미납, 결제확인 필요, 수업완료 미처리, 회원연결 승인, 재등록 대상이 표시된다.
- 주간 시간표 상세 화면은 7일 열 비교형으로 한 주 전체를 한눈에 보여준다.
- 주간 시간표에서는 슬롯 상태를 `가능`, `요청중`, `확정`, `차단`으로 구분한다.
- 예약 요청은 관장이 승인/거절한다.
- 확정 수업은 관장이 `수업완료`를 눌러야 1회 차감된다.
- 24시간 이내 취소요청은 관장이 차감/미차감으로 판단한다.
- 회원 상세에서는 PT권, 잔여횟수, 결제상태, 예약 이력, 연장/환불/차감 이력을 확인한다.
- 관리자 설정에서 운영 정책을 조정한다.
- 관리자 설정에서 회원, PT권, 예약, 결제, 운영 정책 등을 선택해 단일 CSV 파일로 내보낼 수 있다.
- CSV 내보내기는 개인정보 포함 여부를 관리자가 명시적으로 선택해야 한다.

## 5. 회원 기능

- 회원은 본인 PT권의 잔여횟수, 만료일, 결제상태, 재등록 안내를 확인한다.
- 회원은 내역 화면에서 회원권 상세, 결제일/결제상태, 차감/연장/환불 이력, 예약 이력을 확인한다.
- 회원은 공개된 예약 가능 슬롯 중 1개를 예약 요청한다.
- 요청된 슬롯은 정책에 따라 일정 시간 동안 잠긴다.
- 수업 시작 24시간 전까지는 회원이 직접 자동취소할 수 있다.
- 24시간 이내에는 취소요청으로 전환되고 관장이 차감 여부를 결정한다.
- 회원은 연장 요청이 허용된 정책에서 사유와 일수를 입력해 연장을 요청할 수 있다.

## 6. 예약 정책

- 회원에게 기본 4주치 예약 가능 슬롯을 공개한다.
- 회원 예약 요청은 기본 24시간 동안 잠긴다.
- 요청 만료 시 슬롯은 다시 열린다.
- 수업은 `50분 수업 + 10분 정리`로 안내하고, 예약표에서는 60분 슬롯을 점유한다.
- 회원 예약 개수 제한은 정책으로 설정한다.
- 기본 제한은 잔여횟수 기준이다.
- 관장은 필요하면 미납 회원 예약 허용 여부를 정책으로 조정한다.

## 7. 취소와 차감

- 수업 시작 24시간 전까지는 회원 자동취소가 가능하며 차감하지 않는다.
- 24시간 이내 취소/노쇼는 기본 차감한다.
- 질병, 부상 등 예외 사유는 관장이 미차감 처리할 수 있다.
- 차감/미차감 판단은 이력으로 남긴다.

## 8. PT권과 재등록

- PT권 상품은 1회~10회권을 기본으로 둔다.
- 기본 가격은 1회 60,000원이며 현금/카드 동일이다.
- 기본 유효기간은 1~5회권 30일, 6~10회권 60일이다.
- 상품별 회차, 가격, 유효기간, 활성/비활성은 관리자 설정에서 조정 가능하다.
- 재등록 기준은 잔여횟수와 만료일 기준으로 설정한다.
- 기본 재등록 기준은 잔여 2회 이하 또는 만료 7일 전이다.

## 9. 결제와 BOX POS

- BOX POS는 실제 결제 도구로 사용한다.
- 앱은 BOX POS 결제 링크를 직접 발송하지 않고 결제 상태와 이력을 관리한다.
- 결제 상태는 `미납`, `결제요청`, `결제완료`, `환불`로 관리한다.
- PT권은 생성 즉시 활성화할 수 있고, 미납 상태는 관리자/회원 화면에 명확히 표시한다.
- 결제 상태 변경은 누가, 언제, 어떤 근거로 변경했는지 이력으로 남긴다.

## 10. 운영 정책 설정

- 운영 규칙은 관리자 설정에서 최대한 조정 가능하게 한다.
- 단, DB 무결성, 권한, 감사로그, 중복차감 방지는 관리자가 끌 수 없는 고정 안전장치로 둔다.
- 설정 가능 항목:
  - PT 상품: 회차, 가격, 기본 유효기간, 활성/비활성
  - 예약: 공개 주차, 요청 만료 시간, 회원 예약 개수 제한, 미납 예약 허용 여부
  - 취소: 자동취소 기준 시간, 당일취소 기본 차감 여부, 예외 사유 목록
  - 연장: 회원 연장요청 허용 여부, 승인 방식, 기본 사유 목록
  - 재등록: 잔여횟수 기준, 만료일 기준, 회원 노출 여부
  - 문구: 앱 내 알림 문구, 카카오톡 복사용 문구
- 정책 변경은 신규 PT권/신규 예약부터 적용한다.
- 기존 PT권과 예약에는 생성 당시 주요 정책값을 스냅샷으로 저장한다.

## 11. 데이터와 보안

- 핵심 테이블은 `members`, `admin_users`, `member_link_requests`, `pt_pass_products`, `pt_passes`, `pass_events`, `policy_settings`, `availability_templates`, `availability_slots`, `reservations`, `payments`, `payment_events`, `extension_requests`, `notifications`이다.
- 잔여횟수는 현재값만 저장하지 않고 `pass_events`로 차감/복구/연장/환불 이력을 남긴다.
- 결제는 `payment_events`로 상태 변경 이력을 남긴다.
- 회원 화면은 Supabase RLS로 승인된 본인 데이터만 조회 가능하게 한다.
- 관리자 권한은 별도 `admin_users` 기준으로 관리한다.
- 수업완료와 1회 차감은 하나의 DB 트랜잭션으로 처리한다.

## 12. MVP 이후 확장 범위

- 체육관 전체 회원권, 그룹수업, 출석부, 락커, 물품 판매
- 다중 관리자 권한, 복수 지점, 정산 대시보드
- SMS 본인인증, 카카오톡 자동발송 API
- PG 결제 연동, 실제 BOX POS API 연동
- 비타민CRM 자동 연동
